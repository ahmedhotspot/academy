# الملخص العام للنظام وطريقة الاستخدام الصحيحة

## ما الذي يقدمه هذا النظام؟

هذا النظام هو منصة إدارية لإدارة أكاديمية تعليم القرآن الكريم (حضوري) من خلال 3 فروع بإدارة مركزية.

يركز النظام على 3 أدوار رئيسية:
- المشرف العام
- السكرتيرة
- المعلم

الهدف الأساسي: تنظيم العمل اليومي بالكامل داخل الأكاديمية من التسجيل والمتابعة وحتى التحصيل المالي والتقارير.

## الوحدات الأساسية داخل النظام

### 1) إدارة الفروع
- إدارة بيانات الفروع وحالتها
- ربط الطلاب والمعلمين والحلقات بكل فرع

### 2) إدارة المستخدمين والصلاحيات
- إنشاء المستخدمين وربطهم بالأدوار
- الصلاحيات مبنية على `spatie/laravel-permission`

### 3) إدارة الطلاب وأولياء الأمور
- تسجيل الطلاب وربط كل طالب بولي أمر
- حفظ البيانات الأساسية مثل العمر والحالة ووسائل التواصل

### 4) إدارة المستويات والمسارات
- المستويات: مبتدئ، متوسط، متقدم، إجازة
- المسارات: عربي، أعجمي

### 5) إدارة الحلقات والجداول
- إنشاء الحلقات وربطها بالمعلم والفرع والمستوى والمسار
- تحديد نوع الحلقة (فردي/مجموعة) ومواعيدها

### 6) التسجيل والحضور
- تسجيل الطلاب في الحلقات
- حضور المعلمين
- متابعة حضور الطلاب من خلال سجلات المتابعة التعليمية

### 7) المتابعة التعليمية والاختبارات
- سجلات متابعة (حفظ، مراجعة، تجويد، تدبر، أخطاء متكررة، إتقان)
- اختبارات أسبوعية وشهرية واختبارات ختم

### 8) الاشتراكات والتحصيل المالي
- خطط رسوم متعددة (شهري، نصف شهري، أسبوعي، بالحلقة)
- اشتراكات الطلاب وربطها بخطط الرسوم
- تسجيل المدفوعات وإصدار أرقام الإيصالات
- متابعة المتأخرات والمديونيات

### 9) مستحقات المعلمين والمصروفات
- مستحقات المعلمين (راتب أساسي + خصومات + جزاءات + مكافآت)
- مصروفات التشغيل (إيجار، كهرباء، صيانة ...)

### 10) التقارير ولوحة التحكم
- Dashboard لعرض مؤشرات تشغيلية وتعليمية ومالية
- تقارير للطلاب، الحضور، التقدم، الاختبارات، الاشتراكات، الرواتب، المصروفات

## طريقة الاستخدام الصحيحة للنظام (تشغيل فعلي)

## 1. تجهيز النظام أول مرة

نفّذ الأوامر التالية من جذر المشروع:

```powershell
composer install
cp .env.example .env
php artisan key:generate
php artisan migrate:fresh --seed
php artisan serve
```

> ملاحظة: إذا كان مشروعك يستخدم `database.sqlite` تأكد أن الملف موجود ومُعرّف بشكل صحيح في `.env`.

## 2. تسجيل الدخول

- ادخل على شاشة تسجيل الدخول
- استخدم حسابًا حسب الدور المطلوب للتجربة (مشرف/سكرتيرة/معلم)
- يفضّل اختبار كل دور منفصلًا للتأكد من الصلاحيات

## 3. التسلسل التشغيلي الصحيح يوميًا

لضمان عمل النظام بدون تضارب، اتبع هذا الترتيب:

1. ضبط بيانات الأساس:
   - الفروع
   - المستويات
   - المسارات
   - المستخدمون (المعلمين/السكرتارية)

2. تسجيل البيانات التعليمية:
   - أولياء الأمور
   - الطلاب
   - الحلقات
   - جداول الحلقات
   - تسجيل الطلاب داخل الحلقات

3. التشغيل اليومي:
   - حضور المعلمين
   - متابعة الطلاب التعليمية
   - الاختبارات

4. التشغيل المالي:
   - خطط الرسوم
   - اشتراكات الطلاب
   - المدفوعات والإيصالات
   - مستحقات المعلمين
   - مصروفات التشغيل

5. المتابعة الإدارية:
   - مراجعة Dashboard
   - استخراج التقارير حسب الفترة/الفرع/الحالة

## 4. أفضل ممارسات الاستخدام

- أدخل بيانات الفروع والمعلمين قبل إنشاء الحلقات
- لا تسجل طالبًا في حلقة بدون ربطه بولي أمر
- حدّث الحالة المالية للاشتراك بعد كل دفعة
- راجع تقارير المتأخرات أسبوعيًا
- راجع حضور المعلمين والمتابعة التعليمية بشكل يومي
- قبل العرض التجريبي، نفّذ `migrate:fresh --seed` لضمان بيانات واضحة وحديثة

## 5. ما المتوقع بعد التهيئة الصحيحة؟

- لوحة التحكم تعرض أرقام فعلية
- جداول DataTable تحتوي بيانات جاهزة للتصفية والبحث
- التقارير تعمل مباشرة
- النظام جاهز للعرض التجريبي (Demo)
