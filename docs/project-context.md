# السياق الثابت للمشروع

هذا الملف مرجع ثابت مع `/.github/copilot-instructions.md`.
قبل إنشاء أي كود جديد، يجب الرجوع إلى الملفين معًا والالتزام بما فيهما فقط.

## المتطلبات التقنية المعتمدة

- المشروع Laravel 12.
- architecture بنظام Model-based وليس Modules.
- جميع صفحات الإدارة داخل `resources/views/admin`.
- كل صفحات Blade تستخدم:

```blade
@extends('admin.layouts.master')
```

- لا تضع أي validation داخل Blade نهائيًا.
- استخدم Form Requests فقط.
- جميع النصوص والرسائل والعناوين والأزرار بالعربي فقط.
- جميع صفحات index تعمل بـ DataTable Ajax.
- الصفحات المطلوبة دائمًا: index / create / edit / show.
- الصفحات تكون احترافية ومنظمة وجذابة.
- Controllers تكون thin.
- business logic داخل Actions أو Services.
- route names تحت admin.*.
- prefix يكون /admin.
- استخدم spatie/laravel-permission للصلاحيات.

## ملخص النشاط الفعلي

- مؤسسة غير ربحية لتعليم القرآن الكريم.
- تعمل من خلال 3 فروع بإدارة مركزية.
- بها حوالي 150 طالب.
- بها 14 معلم ومعلمة.
- يوجد سكرتيرتان.
- يوجد موظفو خدمات لكن النظام الإداري الحالي يركز على: المشرف العام + السكرتيرة + المعلم.
- الدراسة حضوري فقط.
- الفئات العمرية من 5 سنوات فأكثر.
- كلا الجنسين.
- النظام فردي ومجموعات.
- يوجد تصنيف للطلبة: مبتدئ / متوسط / متقدم / إجازات.
- يوجد منهج عربي ومنهج أعجمي.
- يوجد متابعة: الحفظ / المراجعة / التجويد / التدبر / الأخطاء المتكررة / الإتقان.
- يوجد اختبارات أسبوعية وشهرية وعند ختم جزء.
- يوجد حضور وغياب للطلاب والمعلمين.
- يوجد رسوم واشتراكات: شهري / نصف شهري / أسبوعي / بالحلقة.
- يوجد خصومات للأخوات.
- يوجد متأخرات ومديونيات.
- يوجد إيصالات قبض.
- يوجد مصروفات تشغيل.
- يوجد مستحقات معلمين تشمل: الاستقطاعات / الجزاءات / المكافآت.
- يوجد إشعارات وتقارير.
- النظام المطلوب: ويب + موبايل.
- اللغة: عربي فقط.
- مطلوب: استيراد وتصدير Excel + طباعة تقارير + نسخ احتياطي.
